# 电商属性库构建 Skill

## 目标
构建标准化电商属性体系

## 目标规模
- **1000+ 属性项** (第一阶段)
- 10,000+ 属性值
- 覆盖80%场景

## 属性项设计 (第一阶段: 1000+)

### 顶层分类
1. **基本信息** - 品牌、型号、名称、描述...
2. **价格信息** - 价格、折扣、优惠...
3. **品类信息** - 大类、中类、小类...
4. **规格参数** - 尺寸、重量、容量...
5. **外观属性** - 颜色、材质、样式...
6. **功能特性** - 功能、特性、参数...
7. **产地物流** - 产地、物流、库存...
8. **商家信息** - 店铺、信誉、服务...
9. **用户评价** - 评分、评论、问答...
10. **其他属性** - 保修、证书、附件...

### 属性值设计
每个属性项下的典型值：
- 品牌：Apple、Samsung、Nike...
- 颜色：红、蓝、黑、白...
- 材质：棉、丝绸、聚酯纤维...
- 尺寸：S、M、L、XL...

## 属性关系

### 上下位关系
- 品类→具体品类 (手机→智能手机→iPhone)

### 互斥关系
- 颜色之间互斥

### 依赖关系
- 规格依赖品类

## 输出格式

```json
{
  "attributes": [
    {
      "id": "brand",
      "name": "品牌",
      "type": "enum",
      "values": [...],
      "relations": []
    }
  ]
}
```

## 覆盖率评估
完成后需要评估：
- 当前属性项覆盖了多少商品？
- 是否需要扩展到5000或10000？
