# 属性中台自演进系统 Skill

## 目标
建立属性库与上游业务的持续反馈闭环

## 系统架构

```
┌─────────────┐    需求    ┌─────────────┐
│  用户意图   │ ────────→ │            │
├─────────────┤    需求    │            │
│    场景     │ ────────→ │  需求收集   │
├─────────────┤    需求    │            │
│  运营选品   │ ────────→ │            │
├─────────────┤    需求    └──────┬─────┘
│    搜索     │ ──────────────────────┤
├─────────────┤                          │
│    推荐     │ ←───────────────────────┘
└─────────────┘    属性供给
```

## 核心模块

### 1. 需求收集器
```json
{
  "business": "搜索业务",
  "requirement": "需要按价格筛选",
  "required_attributes": ["价格", "折扣", "促销价"],
  "priority": "高",
  "status": "待处理"
}
```

### 2. 差距分析器
- 需求 → 需要属性 → 现有属性 → 差距
- 输出：差距列表

### 3. 属性补充器
- 根据差距补充属性
- 更新属性库

### 4. 效果验证器
- 业务效果是否提升
- 反馈到系统

## 迭代流程

1. 收集上游业务需求
2. 分析差距
3. 补充/优化属性
4. 验证效果
5. 持续迭代

## 数据结构

### demand_registry.json
```json
{
  "demands": [
    {
      "id": "搜索_价格筛选",
      "business": "搜索业务",
      "requirement": "按价格筛选商品",
      "required_attributes": ["价格", "折扣"],
      "priority": "P0",
      "status": "已满足/待处理/进行中"
    }
  ],
  "attribute_coverage": {
    "搜索业务": {"需求数": 10, "已满足": 8, "覆盖率": "80%"}
  }
}
```

## 实施步骤

1. 初始化需求注册表
2. 映射现有需求到属性
3. 识别差距
4. 补充缺失属性
5. 建立反馈机制
