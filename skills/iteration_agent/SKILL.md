# Project Iteration Agent Skill - 24/7项目迭代

## 目标
创建一个24小时运行的后台Agent，使用Codex迭代优化项目，每次迭代要有明确的增量。

## 核心功能

### 1. 增量迭代（主动）
按照迭代流程执行：
1. 检查项目状态
2. 分析待解决问题
3. 制定增量计划
4. 执行迭代
5. 测试验证
6. 提交推送
7. 记录日志

### 2. 空闲自动迭代（被动）
- **监控频率**: 每5分钟检查一次Claude调用量
- **触发条件**: 2小时内Claude调用量 < 100次
- **执行操作**: 自动开始增量迭代

### 3. 用户控制
- 用户发送明确消息时，进入空闲状态
- 等待用户下一个明确指令才继续

## 监控逻辑
```
循环：
  检查用户消息（阻塞等待）
  
  如果收到用户消息：
    处理用户指令
    如果是"停止"或"空闲"：
      进入空闲等待状态
  
  否则（无用户消息）：
    检查Claude使用量
    如果2小时内使用量 < 100：
      开始增量迭代
```
