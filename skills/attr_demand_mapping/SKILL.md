# 属性需求映射系统 Skill

## 核心问题
属性库是商品库中台，但上游业务的需求与属性供给的映射关系尚未建立。

## 上游业务需求分析

### 1. 用户意图业务
**需要什么：**
- 搜索意图理解
- 用户 Query 解析
- 意图分类

**需要什么属性：**
- 品类属性（用户想买什么）
- 价格属性（用户预算）
- 品牌属性（用户偏好）
- 规格属性（用户需求的具体参数）

### 2. 场景业务
**需要什么：**
- 使用场景匹配
- 送礼推荐
- 自用推荐

**需要什么属性：**
- 适用场景属性
- 送礼对象属性
- 适用人群属性
- 节日属性

### 3. 运营选品业务
**需要什么：**
- 爆款识别
- 利润款识别
- 引流款识别
- 新品筛选

**需要什么属性：**
- 销量属性
- 价格带属性
- 利润率属性
- 复购率属性
- 新品期属性

### 4. 搜索业务
**需要什么：**
- 召回匹配
- 排序优化
- 筛选项配置

**需要什么属性：**
- 可筛属性
- 可排序属性
- 搜索热度属性
- 相关性属性

### 5. 推荐业务
**需要什么：**
- 用户画像
- 商品画像
- 协同过滤
- 精准推荐

**需要什么属性：**
- 用户偏好属性
- 商品相似度属性
- 行为数据属性
- 互补/替代商品属性

## 需求-属性映射矩阵

```
业务需求 | 需求描述 | 需要的属性 | 现有属性 | 差距
--------|----------|------------|----------|----
搜索召回 | 品类匹配 | 品类ID | 有 | 无
价格筛选 | 价格区间 | 价格、折扣 | 有 | 无
场景推荐 | 送礼场景 | 适用场景 | 无 | 需要新增
爆款识别 | 高销量商品 | 销量 | 部分 | 需要完善
```

## 自演进机制

### 1. 需求收集
- 业务方提交需求
- 自动识别需要什么属性

### 2. 差距分析
- 需求 → 需要属性 → 现有属性 → 差距

### 3. 属性补充
- 根据差距补充属性

### 4. 效果验证
- 补充后业务效果是否提升

## 输出
1. 需求-属性映射矩阵
2. 差距分析报告
3. 属性补充优先级
