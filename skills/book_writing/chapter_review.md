# Chapter Review Skill - 章节复查指南

## Purpose
Review each completed chapter of the book "供给理解及其评测体系" for quality, completeness, and consistency.

## Review Checklist

### 1. 内容完整性
- [ ] 所有小节是否都已覆盖？
- [ ] 核心概念是否都有定义？
- [ ] 是否有遗漏的重要主题？

### 2. 学术规范性
- [ ] 引用格式是否统一（作者, 年份）？
- [ ] 引用数量是否足够（每章至少15-40篇）？
- [ ] 是否有正确的学术语气？

### 3. 技术深度
- [ ] 算法描述是否足够详细？
- [ ] 架构图是否清晰？
- [ ] 实现细节是否充分？

### 4. 逻辑连贯性
- [ ] 章节之间是否连贯？
- [ ] 段落之间是否衔接自然？
- [ ] 论证逻辑是否完整？

### 5. 创新性
- [ ] 是否有独到见解？
- [ ] 是否有原创性框架？
- [ ] 是否有实践洞察？

### 6. 与其他章节一致性
- [ ] 术语使用是否一致？
- [ ] 符号体系是否统一？
- [ ] 章节之间引用是否正确？

### 7. 格式规范
- [ ] Markdown格式是否正确？
- [ ] 标题层级是否规范？
- [ ] 参考文献格式是否统一？

## Review Process
1. 通读全文，把握整体结构
2. 对照检查清单逐项核对
3. 标记需要修改的具体位置
4. 提出具体修改建议
5. 输出复查报告

## Output Format
输出复查报告，包含：
- 总体评价
- 主要优点
- 需要修改的问题（按优先级排序）
- 具体修改建议
- 建议是否进入下一章

## 复查标准
- 通过：可以直接进入下一章
- 小修：需要小幅修改后可进入下一章
- 大修：需要重大修改后重新复查
