# 图3.2：TransE模型原理

```
TransE核心思想： h + r ≈ t

头实体 + 关系 ≈ 尾实体

示例：
┌─────────────────────────────────────────────────────────────┐
│ 三元组: (iPhone15, is_a, 智能手机)                         │
│                                                             │
│  向量表示:                                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  h(iPhone15) = [0.2, -0.1, 0.5, ...]               │   │
│  │  r(is_a)     = [0.1, 0.3, -0.2, ...]              │   │
│  │  ─────────────────────────────────────              │   │
│  │  h + r     ≈ [0.3, 0.2, 0.3, ...]                  │   │
│  │  t(智能手机) ≈ [0.3, 0.2, 0.3, ...]                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  得分函数:                                                  │
│  f(h,r,t) = -||h + r - t||₂                               │
│                                                             │
│  正例三元组: 得分高 (h+r≈t)                                │
│  负例三元组: 得分低 (h+r≠t)                                │
└─────────────────────────────────────────────────────────────┘

TransE的局限性:
┌─────────────────────────────────────────────────────────────┐
│  1-1关系: ✓ 工作良好                                        │
│  (中国-首都-北京)                                           │
│                                                             │
│  1-N关系: ⚠ 可能有问题                                      │
│  (iPhone15-is_a-手机)                                      │
│  (三星S23-is_a-手机)                                        │
│  → iPhone15和三星S23向量会非常接近                         │
│                                                             │
│  N-1关系: ⚠ 可能有问题                                      │
│  (多个手机-is_a-电子设备)                                  │
└─────────────────────────────────────────────────────────────┘
```
