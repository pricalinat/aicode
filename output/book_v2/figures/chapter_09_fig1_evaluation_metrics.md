# 图9.1：服务评测指标体系

## 评测维度与指标设计

本章介绍了完整的服务评测指标体系，涵盖效果、性能、安全、成本四大维度：

| 维度 | 核心指标 | 衡量目标 |
|------|---------|----------|
| 效果 | 准确率、召回率、F1、NDCG | 模型输出质量 |
| 性能 | 延迟、吞吐量、并发数 | 响应能力 |
| 安全 | 风险拦截率、合规通过率 | 内容安全 |
| 成本 | 单次调用成本、ROI | 商业可行性 |

```mermaid
flowchart TD
    subgraph 评测体系
        A[服务评测] --> B[效果评测]
        A --> C[性能评测]
        A --> D[安全评测]
        A --> E[成本评测]
    end
    
    subgraph 效果评测
        B --> B1[准确率]
        B --> B2[召回率]
        B --> B3[F1分数]
        B --> B4[NDCG]
        B --> B5[Bleu/ Rouge]
        B --> B6[Perplexity]
    end
    
    subgraph 性能评测
        C --> C1[P50延迟]
        C --> C2[P99延迟]
        C --> C3[吞吐量]
        C --> C4[并发能力]
        C --> C5[首Token时间]
    end
    
    subgraph 安全评测
        D --> D1[风险拦截率]
        D --> D2[合规通过率]
        D --> D3[拒答率]
        D --> D4[误杀率]
        D --> D5[安全评分]
    end
    
    subgraph 成本评测
        E --> E1[单次调用成本]
        E --> E2[Token成本]
        E --> E3[计算资源成本]
        E --> E4[人工审核成本]
        E --> E5[ROI]
    end
    
    F[综合评分] --> B
    F --> C
    F --> D
    F --> E
```
