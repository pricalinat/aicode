# 图4.3：层次分类方法

```
层次类目结构:
┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│                         电子产品                                      │
│                        /        \                                    │
│                   智能手机          电脑                             │
│                  /     \           /    \                           │
│              5G手机   4G手机    轻薄本  游戏本                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘

方法一: 逐层分类 (Hierarchical Classification)
┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Level 1: 电子产品? → Yes                                           │
│       ↓                                                              │
│  Level 2: 智能手机? → Yes                                           │
│       ↓                                                              │
│  Level 3: 5G手机? → Yes                                            │
│       ↓                                                              │
│  预测结果: [电子产品 → 智能手机 → 5G手机]                          │
│                                                                      │
│  P(5G手机|x) = P(电子产品|x) × P(智能手机|电子产品,x)             │
│                     × P(5G手机|智能手机,x)                          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘

方法二: 层次约束分类
┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  • 预测时加入层次约束                                              │
│  • 确保父类预测为Yes时，子类才能预测为Yes                          │
│  • 避免出现"手机→电脑"的跨层错误                                   │
│                                                                      │
│  约束示例:                                                          │
│  If P(电子产品|x) < threshold:                                      │
│      P(智能手机|x) = 0                                             │
│      P(电脑|x) = 0                                                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```
